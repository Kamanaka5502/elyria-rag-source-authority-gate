# Contributing

## Public-Safe Contributions Only

Contributions should strengthen the public-safe enterprise RAG source authority model.

Accepted contribution types:

- documentation improvements
- public-safe RAG source scenarios
- clearer decision logic explanations
- sandbox usability improvements
- source authority checklist improvements
- public-safe expected outcomes
- buyer or pilot-readiness templates

Do not contribute:

- credentials
- customer data
- private client artifacts
- protected Elyria Systems runtime machinery
- confidential diagrams
- proprietary implementation internals
- production logs
- regulated data

## Review Standard

A contribution should preserve the repository purpose:

```text
Govern what an AI system may retrieve, trust, cite, and use before output becomes enterprise consequence.
```

## Decision Language

Use the repository decision model consistently:

```text
ADMIT / HOLD / REFUSE / REVALIDATE
```
