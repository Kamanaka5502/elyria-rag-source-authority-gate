# Design Language

## Purpose

This document defines the public-facing visual and language system for the Elyria RAG Source Authority Gate.

The goal is polished enterprise flair: visually sharp, executive-readable, credible for buyers, and still disciplined enough for security, data, privacy, and architecture review.

---

## Visual Positioning

```text
Retrieval trust
Source authority
Grounding readiness
Access-controlled knowledge
Evidence-backed output
Pilot-ready review
Enterprise-safe RAG
```

---

## Palette

| Role | Color | Use |
|---|---|---|
| Deep graphite | `#1f4f5a` | Primary architecture, control plane, trust boundary |
| Signal teal | `#2f6f73` | Approved sources, ADMIT, grounding readiness |
| Muted steel | `#5f8fa3` | Telemetry, observability, platform evidence |
| Warm gold | `#c9a66b` | HOLD, review, readiness gates, executive emphasis |
| Refusal red | `#b94b4b` | REFUSE, unauthorized access, blocker state |
| Electric cyan | `#62d9ff` | Hero visuals, retrieval flow, source graph energy |

---

## Badge System

Use badges to make the repo scan immediately as an enterprise asset:

```text
RAG Governance
Source Authority
Grounding Readiness
Access Scope
Deployable Sandbox
Executive Ready
Pilot Ready
Telemetry Evidence
```

Badges should clarify capability, not decorate without meaning.

---

## Voice

Use language that is:

- precise
- enterprise-readable
- buyer-facing
- architecture-oriented
- governance-forward
- public-safe
- confident without overclaiming

Avoid claiming that this public repo is a complete production enforcement system. The accurate claim is stronger:

```text
public-safe enterprise reference architecture and deployable pilot sandbox
```

---

## Core Phrases

```text
RAG retrieval is not source authority.
Govern what AI may retrieve, trust, cite, and use.
Turn RAG from document retrieval into governed source authority.
Approved sources. Authorized access. Fresh content. Grounded output. Evidence preserved.
ADMIT / HOLD / REFUSE / REVALIDATE.
Show the architecture. Protect the machinery.
```

---

## Decision Signal Style

| Decision | Visual Signal | Meaning |
|---|---|---|
| ADMIT | Teal / green | Source authority is resolved. |
| HOLD | Gold | Evidence, freshness, ownership, or grounding is incomplete. |
| REFUSE | Red / graphite | Source approval, access, or sensitivity boundary is missing. |
| REVALIDATE | Steel / cyan | Conditions changed; approval must be reviewed again. |

---

## Public Boundary Rule

The repo should feel premium, but never expose private Elyria Systems machinery, protected validators, customer builds, confidential source data, commercial proof-corridor internals, credentials, or production secrets.
